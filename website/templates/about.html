Predicting a Startup's Acquisition Status

Summary

The data contains industry trends, investment insights and individual company information. Since the data was acquired on a trial basis, it only contains information about companies. After training the model, we predict whether startups still operating, IPO, acquired, or closed.

Objective

Company's Status Prediction: Binary And Multiclass Classification Approach

The objective of the project is to predict whether a startup which is currently Operating, IPO, Acquired, or closed.This problem will be solved through a Supervised Machine Learning approach by training a model based on the history of startups which were either acquired or closed.

Features

Dataset provides Company name, permalink, category, funding dates, funding rounds, funding amount, city, state, funding dates, last milestone date, etc.

Dataset

Link to raw data(Huge JSON and Excel file)

Machine Learning Model Development:

  • Data Collection: Gathered detailed information about startups from Crunchbase.com. a comprehensive database of companies and startup ecosystem. Utilizing Crunchbase's extensive dataset, we collected company profiles, funding rounds, investment insights, industry trends, and other relevant information. This included extracting data points such as company names, industry categories, funding amounts, funding dates, geographic locations, and other key metrics. The data from Crunchbase provided valuable insights into the startup landscape, allowing us to build a robust predictive model for determining a startup's acquisition status.
  • Data Cleaning: Performed data cleaning tasks such as handling missing values, removing duplicates, and dealing with outliers to ensure data quality.
  • Exploratory Data Analysis (EDA): Conducted EDA to gain insights into the dataset, identify patterns, and explore relationships between variables.
  • Data Preprocessing: Prepared the data for modeling by scaling numerical features, encoding categorical variables, and splitting the dataset into training and testing sets.
  • Feature Engineering: Created new features or transformed existing features to better represent the underlying data and improve model performance.
  • Handling Imbalanced Dataset: Addressed imbalanced class distribution by applying techniques such as SMOTE (Synthetic Minority Over-sampling Technique) and random oversampling.
  • Mutual Information Scores for Feature Importance Analysis: Calculated mutual information scores to analyze the importance of features in predicting the target variable.
  • Feature Extraction: Extracted relevant features from the dataset to improve model performance and reduce dimensionality.
  • Model Selection: Explored different machine learning algorithms suitable for the task such as logistic regression, decision trees, random forests, XGB, KNN and Gradient Boosting.
  • Model Training: Trained multiple models using the training data and evaluated their performance using appropriate metrics.
  • Hyperparameter Tuning: Fine-tuned the hyperparameters of the best-performing models using techniques like grid search or randomized search.
  • Model Evaluation: Evaluated the final model on the testing data to assess its generalization performance and identify any potential issues like overfitting or underfitting.
  • Model Deployment: Deployed the trained model into production environment for real-time predictions.
